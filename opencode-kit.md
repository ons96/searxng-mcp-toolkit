# OpenCode Kit — Portable Setup Manifest

OpenCode plugins + MCP tools + config for a low-resource machine (i3-3217U, 6GB).

## Repo

All setup files live in a private GitHub repo (`opencode-setup`).
Clone to `~/.config/opencode/setup/` on each machine:

```bash
git clone git@github.com:YOUR_USER/opencode-setup.git ~/.config/opencode/setup
```

Local cron keeps instance data fresh:
```cron
0 7 * * * cd ~/.config/opencode/setup && git pull -q
```

## Plugin Manifest

Install in `~/.config/opencode`:

```bash
npm install \
  @tarquinen/opencode-dcp@latest \
  opencode-antigravity-auth@latest \
  opencode-handoff@latest \
  opencode-mystatus@latest \
  @pemessage/opencode-worktree@latest \
  opencode-optimal-model-temps@latest \
  oc-sessions@latest
```

Add to `opencode.json`:
```json
"plugin": [
  "@tarquinen/opencode-dcp@latest",
  "opencode-antigravity-auth@latest",
  "opencode-handoff@latest",
  "opencode-mystatus@latest",
  "@pemessage/opencode-worktree@latest",
  "opencode-optimal-model-temps@latest",
  "oc-sessions@latest"
]
```

## MCP Tools

### SearXNG Web Search (no API key)

Zero-dep Python MCP. Reads pre-sorted instances.json (generated by CI).
Tries JSON instances first, falls through to HTML scrape.

```json
"searxng": {
  "command": ["/usr/bin/python3", "/home/USER/.config/opencode/setup/searxng-mcp.py"],
  "enabled": true,
  "type": "local"
}
```

Update path to match your home dir.

### Brave Search (API key required)

```json
"brave-search": {
  "command": ["npx", "-y", "@brave/brave-search-mcp-server"],
  "enabled": true,
  "environment": {
    "BRAVE_API_KEY": "{file:~/.secrets/brave-api-key}"
  },
  "type": "local"
}
```

### lean-ctx (context compression)

```json
"lean-ctx": {
  "command": ["/path/to/lean-ctx"],
  "enabled": true,
  "type": "local"
}
```

## CI: SearXNG Instance Health

GitHub Actions in `.github/workflows/test-searxng.yml` runs daily:
1. `scrape-searxng.py --merge --ci` — discovers new instances from registries
2. `test-searxng.py --ci` — tests all instances, writes `instances.json` (sorted by speed)
3. Commits and pushes updated data back to repo

The MCP reads `instances.json` — no testing or sorting done locally.

## Files

| File | Purpose |
|------|---------|
| `searxng-instances.yaml` | Master DB (145 instances, full metadata) |
| `instances.json` | CI-generated sorted list (MCP-consumable) |
| `searxng-mcp.py` | Stripped MCP server (stdlib only) |
| `searxng-fallback.py` | CLI search tool (debug/inspection) |
| `test-searxng.py` | Full test suite (`--ci` for non-interactive) |
| `scrape-searxng.py` | Auto-discovery from registries (`--merge`) |
| `opencode-kit.md` | This file — portable setup manifest |

## Security

- `opencode.json` contains API keys — NEVER commit to any repo
- Use `{file:~/.secrets/KEY}` pattern in opencode.json for provider keys
- This repo contains only scripts and public SearXNG instance data
